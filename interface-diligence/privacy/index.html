Artifact Backend Privacy Policy

Effective Date: June 25, 2026

Scope: This Privacy Policy applies to the Interface Diligence Artifact Backend used with Interface Diligence Custom GPT agents. It covers run creation, artifact storage, phase status, and report-rendering support. It does not turn Lex Nova HQ or Interface Diligence into a law firm and does not create an attorney-client relationship.

1. Who operates this backend

The Interface Diligence Artifact Backend is operated by Lex Nova HQ / Interface Diligence.

For privacy requests, use the public contact channel listed on this website or the contact method provided by the workflow operator.

2. What data is processed

When a reviewer uses an Interface Diligence GPT agent, the backend may receive and store:

  • target name, company name, domain, URL, or source identifier;
  • reviewer-provided public source material;
  • pasted text supplied by the reviewer;
  • metadata or extracted text from uploaded public PDF, DOC, or TXT material where supplied through the workflow;
  • generated diligence artifacts, including source discovery, legal cartography, target profile, feature profile, data provenance, exposure registry, challenge gate, final handoff, terminal validation, and renderer payloads;
  • run identifiers, phase status, lock status, timestamps, and workflow logs;
  • API request metadata necessary to operate, secure, debug, and audit the backend.

The workflow is designed for public-footprint diligence and reviewer-authorized material. Reviewers should not submit private, confidential, privileged, personal, sensitive, or restricted material unless they are authorized to do so and understand that it may be stored as part of the run.

3. How data is used

Data is used only to:

  • create and isolate diligence runs;
  • save workflow artifacts under an exact run ID;
  • allow authorized downstream agents to retrieve required artifacts;
  • prevent cross-run contamination;
  • support the portfolio renderer or final report view;
  • debug, audit, secure, and operate the workflow.

The backend does not sell personal data. The backend does not use stored artifacts to train AI models. The backend does not itself call OpenAI, Gemini, or other model APIs for reasoning. Reasoning happens inside the Custom GPT conversation controlled by the reviewer.

4. Third-party infrastructure

The backend may use third-party cloud infrastructure and storage services, including:

  • Google Cloud Run;
  • Google Firestore;
  • Google Drive;
  • Google Sheets;
  • OpenAI Custom GPT Actions.

When a Custom GPT uses an Action, relevant parts of the reviewer’s input or generated artifact may be sent from ChatGPT to this backend so the requested workflow step can be completed.

5. Run isolation

Each run is scoped by an exact run_id. The backend is designed to prevent retrieval by company name, domain name, “latest run,” or similar loose lookup. Artifacts should be read or written only by exact run_id and authorized artifact_name.

6. Access controls

Workflow agents have scoped permissions. Agent 1 may save only source_discovery_handoff and legal_cartography_index. Agent 1 does not have permission to read prior artifacts. Downstream agents may read only the artifacts required for their assigned workflow phase.

7. Retention

Artifacts and run records may be retained for operational, audit, debugging, and report-rendering purposes unless deleted or superseded. A reviewer or authorized operator may request deletion of a run or artifact through the public contact channel listed on this website or the contact method provided by the workflow operator.

8. Security

The backend uses API-key authentication for GPT Action requests and stores artifacts in controlled cloud storage. No system can be guaranteed completely secure, but the workflow is designed around scoped run IDs, agent permissions, and phase-level artifact controls.

9. Reviewer responsibilities

Reviewers must not submit material they are not authorized to process. Reviewers must not submit sensitive personal data, privileged legal material, trade secrets, credentials, payment information, medical information, or other restricted material unless the workflow has been separately approved for that use.

10. Changes to this policy

This Privacy Policy may be updated as the workflow, backend, or deployment changes. The effective date above will be updated when material changes are made.

11. Review-ready disclaimer

This policy is a review-ready operational draft for the Interface Diligence Custom GPT workflow. It is not legal advice and should be reviewed by appropriate local counsel before public commercial use.
